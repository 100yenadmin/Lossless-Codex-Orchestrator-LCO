# LCO Drive Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the bounded, audited `lco drive` dry-run workflow shared by CLI, MCP, and OpenClaw for issue #740.

**Architecture:** Add a dependency-injected workflow runner beside the target adapters, then wire thin CLI and MCP surfaces to that single report contract. The runner uses existing Codex and Claude target controls to mint real dry-run audit packets while keeping all live, GUI, and external mutation paths disabled.

**Tech Stack:** TypeScript, Node.js test runner, existing LCO target adapters, MCP registry, generated OpenClaw manifests.

## Global Constraints

- Dry-run is the default and performs zero live target, GUI, source-store, or external mutations.
- Driver and target namespaces must match exactly.
- Raw objective text, transcript content, local paths, credentials, and audit keys never appear in public output.
- Budget bounds are max turns `1..20`, token budget `100..8000`, timeout `1000..600000ms`, and cost ceiling `0..100 USD`.
- Claude remains dry-run only in 1.6; live proof is Codex-only and sacrificial-target-only in the later QA lane.
- CLI, canonical MCP, compatibility alias, and OpenClaw facade must share one implementation.
